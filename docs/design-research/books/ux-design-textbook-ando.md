## 概要

UX（ユーザーエクスペリエンス）デザインの理論・プロセス・手法を学術的な視点から体系的にまとめた、初の日本語書き下ろし教科書。

ISO 9241-210（HCDプロセス）を軸に、UXデザインを単なるUIデザインとは区別し、**ユーザーの体験全体を設計する**という概念を丁寧に解説している。HCDの国際規格・認知科学・感情理論・実践手法を一冊に統合し、**実務と学術をつなぐ橋渡し**の役割を担う。

初学者向けではなく、**UXデザインの基礎知識をすでに持つ人が、知識を理論・プロセスとつなぎ合わせて深めるための書籍**。

### UXデザインフロー（本書が定義する7段階プロセス）

```
① 利用文脈とユーザー体験の把握
   観察・インタビュー・エスノグラフィで現実を理解する
        ↓
② ユーザー体験のモデル化と体験価値の探索
   KA法・価値マップ・ペルソナで「本質的ニーズ」を抽出する
        ↓
③ アイデアの発想とコンセプトの作成
   UXコンセプトツリー・バリューシナリオで体験価値を言語化する
        ↓
④ 実現するUX・利用文脈の視覚化
   アクティビティ/インタラクションシナリオ・ストーリーボードで具体化する
        ↓
⑤ プロトタイプの反復による詳細化
   ペーパープロトタイプ・Wizard of Ozで素早くフィードバックを得る
        ↓
⑥ 実装レベルの制作物によるUX評価
   ユーザビリティテスト・ヒューリスティック評価・フィールドテストで検証する
        ↓
⑦ 体験価値の伝達と保持のための基盤整備
   デザインガイドライン・コンセプトブックとして組織に残す
   （①〜⑦を反復しながら体験価値の実現に近づける）
```

## UXの定義

### ISO 9241-210 による公式定義

> 「製品、システム、またはサービスの使用から生じる、および/または使用を予測した結果としての人の知覚と反応」

### 本書が強調するUXの3要素

| 要素 | 内容 |
|------|------|
| **主観性** | UXはユーザーの感じ方（感情・態度）であり、製品の客観的品質とは別物 |
| **時間的広がり** | 利用前・中・後・長期にわたる体験全体を指す（UX期間モデル） |
| **体験価値** | 単なる「使いやすさ」を超えた、ユーザーにとっての意味・感情的充足 |

### ユーザビリティ vs UX

| 観点 | ユーザビリティ | UX |
|------|-------------|-----|
| **定義** | ISO 9241-11: 特定ユーザーが目標を達成する際の有効さ・効率・満足の度合い | ISO 9241-210: 利用から生じる感覚・感情・態度・行動の総体 |
| **焦点** | タスク完遂の質・効率 | ユーザーの主観的体験全体 |
| **時間軸** | 利用中（使用時点） | 利用前・中・後・長期累積 |
| **感情の扱い** | 満足度として部分的に含む | 感情・意味・快楽を中心概念とする |
| **関係** | ユーザビリティはUXの一構成要素 | UXはユーザビリティを包摂する上位概念 |

## UX期間モデル

本書が採用する「UXの時間的側面」の4分類（Roto・Vermeeren et al. による研究に基づく）：

```
使用前         使用中          使用後         長期
  │              │               │              │
予期的UX ──→ 瞬間的UX ──→ エピソード的UX ──→ 累積的UX
（期待・想像）  （操作感・感情）  （内省・振り返り）  （総合的評価）
```

| 段階 | 英語名 | 内容 |
|------|--------|------|
| **予期的UX** | Anticipated UX | 使用前の期待・想像・購入前評価 |
| **瞬間的UX** | Momentary UX | 利用中、インタラクションの瞬間に体験する感覚 |
| **エピソード的UX** | Episodic UX | 利用後に体験を内省・振り返った際の評価 |
| **累積的UX** | Cumulative UX | 長期にわたる利用期間全体の総合的評価 |

## HCD（Human-Centered Design）プロセス

### ISO 9241-210 に基づく4ステップ（反復的）

```
① 利用状況の理解と明示
   ユーザー・タスク・環境を具体的に把握する
        ↓
② ユーザーと組織の要求の明示
   ユーザー要件と組織要件を整理する
        ↓
③ 要求を満たす設計解の作成
   プロトタイピング・シナリオベースの設計を行う
        ↓
④ 要求に対する設計の評価
   ユーザビリティテスト・ヒューリスティック評価で検証する
        ↓
   （システムが利用要件を満たすまで①〜④を繰り返す）
```

### HCDの6原則（ISO 9241-210）

1. ユーザー・タスク・環境の明確な理解に基づく設計
2. ユーザーが設計・開発全体を通じて参加
3. ユーザー中心の評価により実施・洗練
4. プロセスの反復
5. ユーザーエクスペリエンス全体に取り組む
6. 学際的なスキルと視点を組み込む

### HCDとUCDの違い

| 観点 | UCD | HCD |
|------|-----|-----|
| **提唱** | 1980年代、Donald Norman | ISO 9241-210（2010年改訂） |
| **対象** | ユーザーとの直接的インタラクション | ユーザー＋ステークホルダー全体（組織・社会・規制）への影響 |
| **範囲** | UIレベルのユーザビリティ確保 | 社会・組織を含む包括的な人間中心の設計 |

## 感情とUXの関係（Norman の3レベル感情モデル）

本書が参照するDonald Normanの「感情デザイン」理論（*Emotional Design*, 2004）：

| レベル | 英語名 | 内容 | UXとの対応 |
|--------|--------|------|-----------|
| **本能レベル** | Visceral | 即時・無意識的反応。見た目・手触り・音への直感的反応 | 予期的UX・第一印象 |
| **行動レベル** | Behavioral | 使用中の喜びと効用。習慣的・操作的行動に関連する感情 | 瞬間的UX・ユーザビリティと密接 |
| **内省レベル** | Reflective | 自己イメージ・満足感・記憶・意味づけ | エピソード的・累積的UX |

内省レベルこそが「体験価値」の根幹であるとされる。

## Hassenzahlのモデルと体験価値

本書が参照するMarc Hassenzahlの Hedonic/Pragmatic モデル：

| 品質 | 内容 | 対応する目標 |
|------|------|------------|
| **実用的品質（Pragmatic）** | 使いやすさ・効率性。タスク達成を支援する能力 | do-goals（〜をする） |
| **快楽的品質（Hedonic）** | 感情・心理的側面。刺激・自己同定・喚起 | be-goals（〜でありたい） |

### 快楽的品質の3次元

- **Stimulation（刺激）**：好奇心・個人の成長・スキル発展への欲求
- **Identification（同定）**：製品を通じた自己表現・他者への価値観の表明
- **Evocation（喚起）**：製品が引き起こす記憶・感情の想起

### Jordanのユーザーニーズ階層

Patrick W. Jordan（*Designing Pleasurable Products*, 2000）による階層モデル。Maslowの欲求階層になぞらえた整理：

```
楽しさ（Pleasure）
        ↑
ユーザビリティ（Usability）
        ↑
機能性（Functionality）
```

本書はこの階層を「UXデザインがユーザビリティの先を目指す根拠」として引用する。

## 著者独自のフレームワーク

### ユーザーモデリング3段層（UX三点セット）

```
価値層（なぜ使うか）  → KA法・価値マップで抽出
行為層（何をするか）  → ジャーニーマップ・タスク分析で整理
属性層（誰が使うか）  → ペルソナで具体化
```

3つを組み合わせて使う「UX三点セット」として実務で活用しやすい形で提示。

### 構造化シナリオ法（3段階シナリオ）

| 段階 | 名称 | 内容 | 抽象度 |
|------|------|------|--------|
| 第1段階 | **バリューシナリオ** | ユーザーにどんな価値を提供するか、どのような利用シーンで価値が発揮されるかを文章化 | 最高（抽象） |
| 第2段階 | **アクティビティシナリオ** | どのような行動を促せば価値を感じられるかを記述。具体的なUIは登場しない | 中 |
| 第3段階 | **インタラクションシナリオ** | UIの構成要素を使った具体的な操作を文章化 | 最低（具体） |

### UXコンセプトツリー

```
UXコンセプト（体験価値）
    → 本質的ニーズ
    → 実現手段
    → キー満足要因（KFS）
    → 具体的提案
```

## 主要な方法論（第4章）

### リサーチ手法

| 手法 | 目的 | 特徴 |
|------|------|------|
| **フィールドリサーチ** | ユーザーの実際の環境で行動を観察・理解する | 文脈の中での自然な行動を捉える |
| **コンテキスチュアル・インクワイアリー** | 使用文脈の中でインタビューと観察を組み合わせる | エスノグラフィの一形態 |
| **ユーザーインタビュー** | ニーズ・目標・行動パターンを深掘りする | 定性的な動機・感情の把握 |
| **フォトエッセイ** | ユーザー自身が写真で体験を記録する | ユーザー視点の豊かな文脈情報 |

### 分析・モデリング手法

| 手法 | 目的 | 特徴 |
|------|------|------|
| **KA法（本質的価値抽出法）** | 定性データから「本質的ニーズ（体験価値）」を抽出する | 安藤が UX リサーチ向けにカスタマイズした独自手法 |
| **価値マップ（バリューグラフ）** | KA法の結果を体系化する | 体験価値の全体像を可視化 |
| **ペルソナ** | ユーザー像を具体化し、チームの共通認識を作る | 定性データから作成・ストーリー性が重要 |
| **タスク分析** | ユーザーのゴール達成プロセスを構造化する | 階層的タスク分析（HTA）など |
| **カスタマージャーニーマップ** | 体験の流れと感情を可視化する | 行為層とエモーションカーブを組み合わせる |
| **上位下位分析** | 行為とその目的・手段の連鎖を構造化する | ラダリングの一手法 |

### KA法の手順

```
① ユーザーの発言・行動を「KAカード」に書き出す
        ↓
② カードをKJ法的にグループ化する
        ↓
③「価値マップ（バリューグラフ）」として体系化する
        ↓
④ 本質的ニーズ（体験価値）として命名する
```

### プロトタイピング手法

| 手法 | 目的 | 特徴 |
|------|------|------|
| **ペーパープロトタイピング** | 低コストで素早くアイデアを具体化する | 紙と鉛筆で十分。失敗のコストが低い |
| **Wizard of Oz（オズの魔法使い）** | バックエンド未実装の段階でシステム動作をシミュレートする | 人間が裏でシステムを操作して疑似体験させる |
| **サービスブループリント** | フロントとバックステージの連携を可視化する | サービスデザインと接続する手法 |

### 評価手法

| 手法 | 目的 | 特徴 |
|------|------|------|
| **ヒューリスティック評価** | 専門家がニールセンの10ヒューリスティックスに照らして問題を特定する | ユーザー不要・低コストで発見率が高い |
| **ユーザビリティテスト** | 実際の使用状況でのエラーと改善点を発見する | 実ユーザー観察。より正確だがコスト・時間がかかる |
| **フィールドテスト** | 実際の利用環境でのUXを評価する | 実文脈での検証 |
| **SEPIA法** | ユーザーのモチベーションを分析する | 自己効力感（SE）×製品関与（PI）の4象限でユーザーを分類 |

## AIデザインプロセスへの関連性

- **UX期間モデルへのAI適用**：予期的UX（購入前のレビュー分析）・累積的UX（長期利用データ）を AI が自動収集・分析するパイプライン
- **KA法の自動化**：インタビュー録音・発言記録から AI が本質的ニーズ（体験価値）を自動抽出するシステム
- **バリューシナリオ生成**：ユーザーリサーチデータを入力として AI が3段階シナリオ（バリュー/アクティビティ/インタラクション）を自動生成するワークフロー
- **HCDプロセスの高速化**：反復的な評価フェーズを AI が自動化し、プロトタイプ→評価→改善のサイクルを短縮
- **Hassenzahlモデルへの対応**：ユーザーレビューやアプリストアデータから「実用的品質」と「快楽的品質」を AI が自動スコアリング
- **AI が代替できない役割**：「どんな体験価値を目指すか（UXコンセプトの設定）」という本質的な問いはユーザーとの共創と人間の判断が不可欠
