## 概要

インタラクションデザインの古典的名著。原著初版は1995年で、その後改訂を重ねてきた「インタラクティブ製品デザインの伝説的バイブル」。**ゴールダイレクテッド・デザイン（Goal-Directed Design）** という独自の設計プロセスを提唱している。

アラン・クーパーは「Visual Basicの父」とも呼ばれ、**ペルソナ（Persona）** 手法を先駆的に提唱したデザイン思想家。16年ぶりとなる日本語版（第4版）は OOUI の著者・上野 学が監訳を担当。第4版ではスマートフォン・タブレット対応のモバイルデザインが強化されている。

## 詳細な章構成（3部構成）

### 第1部：ゴールダイレクテッド・デザイン（6章）

| 章 | 内容 |
|----|------|
| 第1章 | デジタル製品に適したデザインプロセス |
| 第2章 | 問題を理解する：デザインリサーチ（フィールドワーク・インタビュー） |

| 第4章 | ビジョンを設定する：シナリオとデザイン要件 |

### 第2部：振る舞いと形態のデザイン（11章）

プラットフォームを問わず適用できる高度なインタラクションデザインの原則を解説。エラー防止・データ入力・ビジュアル統合など。

### 第3部：インタラクション詳細（4章）

モバイル・デスクトップ・ウェブなど、プラットフォーム固有のインターフェースデザインの原則とコントロール設計。

## 主要な概念

### 3つのモデル

| モデル | 説明 |
|--------|------|
| **実装モデル** | エンジニアによって実装されたプログラムが実際に動作する仕組み |
| **表現モデル（メンタルモデルに近づけるモデル）** | プログラムの動きをユーザーに対して表現するために選ばれた方法 |
| **脳内モデル（メンタルモデル）** | ユーザーがプログラムについて単純化して認識している内容 |

デザインの目標は **表現モデルを脳内モデルに近づけること**。技術モデルベースの設計からユーザーモデルへの移行を訴えている。

### ゴールダイレクテッド・デザイン

ユーザーのゴール（目標）を中心に据えた設計プロセス。プロセスの上流から参加し、ユーザー調査 → ペルソナ定義 → シナリオ作成 → デザイン要件化 という流れでデザインの根拠を構築する。

### ペルソナとシナリオ

- **ペルソナ**：フィールドリサーチで得た行動パターンから合成した架空のユーザー像。感情・動機・行動パターンを具体化する
- **シナリオ（コンテキストシナリオ）**：ペルソナが製品を使う具体的な場面を物語形式で記述。デザイン要件の起点となる
- ペルソナ・ゴール・シナリオの組み合わせが「課題定義 → デザイン → 実装」の流れを滑らかに繋ぐ

## AIデザインプロセスへの関連性

- ゴールダイレクテッドデザインの「ユーザーゴールを最優先にする」考えは、AI がユーザーゴールを推論・優先度付けするシステム設計に応用できる
- 3つのモデル（実装・表現・脳内）の整理は、AI が生成する UI とユーザーのメンタルモデルの齟齬を分析するフレームワークになる
- インタビューデータからペルソナを AI が自動生成する研究が進んでいる
- コンテキストシナリオの自動生成は LLM との相性が高い
