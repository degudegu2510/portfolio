## 概要

デザイン・HCI（Human-Computer Interaction）・認知科学の世界的名著。日用品から UI デザインまで、「なぜ人はミスをするのか」「良いデザインとは何か」を認知科学の視点から解説する。

著者は Apple Advanced Technology Group に勤めた経験を持つ認知科学者で、現在は Nielsen Norman Group の共同創業者として活躍。「デザインの世界で最も影響力のある人物」の一人。

## 主要な概念

### アフォーダンスとシグニファイア

| 概念 | 定義 |
|------|------|
| **アフォーダンス** | 対象物と人間との間のインタラクションの**可能性そのもの**（例：椅子は「座れる」というアフォーダンスを持つ） |
| **シグニファイア** | アフォーダンスの存在をユーザーに知覚させる**サイン・手がかり**（例：ドアのプレートは「押す」というシグニファイア） |

初版では「アフォーダンス」と「シグニファイア」を混同していたが、改訂版で明確に区別された。

### 行為の7段階モデル

人間の行為を以下の7段階に分解する：

```
ゴール（Goal）
  ↓ [実行フェーズ]
プラン（Plan）
  ↓
詳細化（Specify）
  ↓
実行（Perform）
  ↓ [評価フェーズ]
知覚（Perceive）
  ↓
解釈（Interpret）
  ↓
比較（Compare）
```

**実行のへだたり**（何をすれば良いか分からない）と**評価のへだたり**（結果が何を意味するか分からない）が UI の問題を引き起こす。

### 6つのデザイン原則

| 原則 | 説明 |
|------|------|
| **発見可能性** | 操作可能なものを見つけられるか |
| **フィードバック** | 操作に対する反応の明示 |
| **概念モデル** | 製品の仕組みについてユーザーが持つ理解（簡素化された地図） |
| **アフォーダンス** | 操作の可能性 |
| **シグニファイア** | 操作の手がかり |
| **対応づけ（マッピング）** | コントロールと結果の自然な対応関係 |

### ヒューマンエラーの分類

| 種類 | 説明 | 対処 |
|------|------|------|
| **スリップ** | 正しい意図を持ちながら操作を誤る | フィードバック強化・制約の追加 |
| **ミステイク** | そもそも間違った意図・計画を持つ | 概念モデルの改善 |

本書の中心的主張：「ヒューマンエラーの多くはデザインの問題である」

## AIデザインプロセスへの関連性

- 行為の7段階モデルは会話型 AI・エージェント AI のインタラクション設計フレームワークとして直接応用できる
- 「実行のへだたり」「評価のへだたり」はエラー設計・リカバリー設計の指針になる
- AI システムのエラー（幻覚・誤動作）をユーザーに適切に伝えるシグニファイア設計が重要
- 概念モデルの整合性を保つという考え方は、AI が生成する UI の一貫性評価に活用できる
