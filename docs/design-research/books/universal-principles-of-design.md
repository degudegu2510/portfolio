## 概要

分野を横断したデザインの普遍的な原則・ガイドライン・ヒューリスティクスを辞書形式でまとめた**クロスディシプリナリー・リファレンス**。グラフィックデザイン・プロダクトデザイン・UI/UX・建築・教育など、あらゆるデザイン領域に適用できる。

各原則は**見開き2ページ**で構成され、左ページに解説・右ページに実例写真・図解という統一フォーマット。辞書として引くことを前提にした設計になっている。

## 収録されている主要な原則（詳細）

### 認知・知覚に関する原則

| 原則 | 概要 |
|------|------|
| **ゲシュタルト原則** | 人間が視覚情報をグループとして認識する法則群。近接・類似・連続・閉合・共通運命・プレグナンツの法則 |
| **チャンキング（Chunking）** | 情報を意味ある塊（チャンク）に分けることで認知負荷を下げる |
| **ミラーの法則** | 短期記憶に保持できる情報量は 7±2 チャンク（George Miller, 1956） |
| **漸進的開示（Progressive Disclosure）** | 情報を段階的に提示し、ユーザーが準備できた時に詳細を提供する |
| **シリアル効果（Serial Position Effect）** | リストの最初と最後の項目が最も記憶に残る |

### 操作・インタラクションに関する原則

| 原則 | 概要 |
|------|------|
| **フィッツの法則** | ターゲットへの移動時間は距離に比例し、サイズに反比例する。`MT = a + b × log₂(2D/W)` |
| **ヒックの法則** | 選択肢の数が増えると意思決定時間が対数的に増加する。`RT = a + b × log₂(n)` |
| **ミラーの法則** | 短期記憶は 7±2 の情報量しか保持できない |
| **GOMS モデル** | タスクをゴール・オペレーター・メソッド・選択ルールで分析する |

### ビジュアルデザインに関する原則

| 原則 | 概要 |
|------|------|
| **シグナル対ノイズ比** | 有用な情報とノイズの比率を最大化する |
| **コントラスト** | 要素間の差を大きくして視認性を高める |
| **整列（Alignment）** | 要素を整列させることで秩序と関係性を伝える |
| **赤ちゃん顔効果** | 幼い顔の特徴を持つものは信頼・親しみを誘発する |
| **ゴールデンレシオ** | 1:1.618の比率がバランスの良いレイアウトを作る |

### デザイン判断・意思決定に関する原則

| 原則 | 概要 |
|------|------|
| **80/20ルール（パレートの法則）** | 20%の原因が80%の結果を生む。20%の機能が80%の使用を占める |
| **オッカムの剃刀** | 余計な要素を削ぎ落とした最もシンプルな解が優れる |
| **KISS 原則** | Keep It Simple, Stupid。シンプルさを最優先にする |
| **レスイズモア** | 少ない要素で多くを表現する |
| **可用性バイアス** | 想起しやすい例に引っ張られる認知バイアス |

### 行動・学習に関する原則

| 原則 | 概要 |
|------|------|
| **IKEA 効果** | 自分で作ったものを過大評価する傾向 |
| **フロー状態** | 難易度とスキルのバランスが取れた時に生まれる最適体験 |
| **スキャフォールディング** | 初期は手厚いサポートを提供し、習熟に応じて減らす |
| **テスト効果** | 思い出そうとする行為自体が記憶を強化する |

## 使い方

「辞書」として使うのが基本：
1. 設計中に問題が生じたとき、関連する原則を引く
2. 設計の根拠を言語化したいときに、対応する原則を探す
3. 各原則の「参考文献」から研究の起点にする

## AIデザインプロセスへの関連性

- 200の原則を AI がリアルタイムで評価・適用する自動デザインレビューシステム
- ユーザーテスト結果を原則に照らして AI が問題を分類するワークフロー（例：「このエラーはヒックの法則の問題」）
- AI が生成した UI を本書の原則セットで自動スコアリング・改善提案
- フィッツの法則・ヒックの法則など定量化可能な原則は AI による UI 最適化に直接適用できる
- LLM のプロンプト設計にも「チャンキング」「漸進的開示」「シグナル対ノイズ比」の考え方が応用できる
