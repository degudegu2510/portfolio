## 概要

**リーンスタートアップ**・**アジャイル開発**・**UXデザイン** の3つの思想を統合した設計プロセス論。「Build-Measure-Learn（作る→測る→学ぶ）」のフィードバックループを UX デザインに適用する。

第3版では「**Lean UX Canvas**」が新たに導入され、チーム全体で製品の戦略的前提を一枚で共有できるようになった。

## 4部構成と詳細な章

| 部 | 内容 |
|----|------|
| 第I部 | イントロダクションと基本原則。Lean UX の重要性・原則・成果の理解 |
| 第II部 | プロセス。Lean UX Canvas・8つのボックス・仮説の立て方と検証 |
| 第III部 | コラボレーション。コラボレーティブデザイン・ユーザーリサーチ・アジャイルとの融合 |
| 第IV部 | 組織での実践。組織変革・エージェンシーでの適用方法 |

## Lean UX Canvas（第3版の核心）

8つのボックスで製品戦略を一枚のキャンバスに可視化するツール：

| ボックス | 内容 |
|----------|------|
| **ビジネスプロブレム** | 解決したいビジネス上の問題 |
| **成果（アウトカム）** | ビジネスが達成すべき測定可能な変化 |
| **ユーザー** | ターゲットとなるユーザーセグメント |
| **ユーザーの成果** | ユーザーにとっての価値・変化 |
| **ソリューション** | 問題を解決するためのアイデア |
| **仮説** | 上記を組み合わせた検証可能な仮説文 |
| **学習事項** | 実験で得た知見 |
| **MVP と実験** | 仮説を検証するための最小限の実験計画 |

## アウトプット・アウトカム・インパクトの区別（第3章）

| 概念 | 定義 | 例 |
|------|------|---|
| **アウトプット** | 作成したもの（機能数・画面数） | 「ログイン画面を作った」 |
| **アウトカム** | 行動の変化・測定可能な結果 | 「新規登録数が20%増加した」 |
| **インパクト** | ビジネス全体への影響 | 「売上が拡大した」 |

Lean UX は**アウトプットではなくアウトカムを成果指標**にすることを強く推奨する。

## 仮説の立て方（第II部）

```
「私たちは [ユーザー] が [問題/目標] を達成できるよう [ソリューション] を作ることで、
[アウトカム] が得られると信じている。
これは [指標] が [変化] することで確認できる」
```

このフォーマットで仮説を文章化し、チームの前提を明示して検証可能にする。

## 3つの思想的基盤

| 基盤 | 説明 |
|------|------|
| **ユーザーエクスペリエンスデザイン** | ユーザーの行動・感情・文脈を中心にした設計 |
| **アジャイルソフトウェア開発** | 短いサイクルで仮説を検証しながら開発する手法 |
| **リーンスタートアップ（Eric Ries）** | MVP を用いた仮説検証ループによるリスク最小化 |

## AIデザインプロセスへの関連性

- Lean UX Canvas の8つのボックスは AI が仮説・実験・学びを整理する構造として活用できる
- アウトカム指標の自動モニタリングと学習への AI 活用
- 仮説文の自動生成・評価を LLM が担う実験が始まっている
- AI による高速プロトタイピング（v0・Figma AI など）は Lean UX の「MVP をすばやく作る」原則を加速する
